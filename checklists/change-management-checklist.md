# Change Management Checklist

This checklist ensures all required steps are followed during the change management process — from planning to closure.

---

## Pre-Implementation
- [ ] Define change objective and scope
- [ ] Classify change (Standard, Normal, Emergency)
- [ ] Identify impacted systems, users, and services
- [ ] Assess risk level using the risk matrix
- [ ] Document implementation, validation, QA, rollback, and monitoring plans
- [ ] Obtain manager and stakeholder approvals
- [ ] Submit change record in the change management system (e.g., Jira Service Management)
- [ ] Schedule CAB review (if applicable)

---

## Implementation Phase
- [ ] Notify impacted teams of planned change
- [ ] Confirm backups or system snapshots are taken
- [ ] Execute change according to the implementation plan
- [ ] Validate change is working as intended
- [ ] Monitor system performance or alerts
- [ ] Document any deviations or unexpected behavior

---

## Post-Implementation
- [ ] Mark change record as “Implemented” or “Closed”
- [ ] Attach change evidence (e.g., logs, screenshots, results)
- [ ] Conduct post-implementation review (PIR)
- [ ] Record lessons learned (optional)
- [ ] Update documentation and runbooks if needed

---

## Governance & Audit
- [ ] Ensure change was linked to a ticket or request
- [ ] Verify approvals are captured in the record
- [ ] Archive or retain change logs per policy
- [ ] Flag incomplete or unauthorized changes for review
